# KNX Optional Source Evidence Pack Prototype

Last updated: 2026-06-01

## Purpose

This folder contains local prototype tooling for the KNX optional source/evidence pack.

It is not part of the KNX governance core.

## Current Tooling

- `scripts/validate_source_evidence.py`: Python standard-library validator for KNX synthetic fixture packs.
- `tests/test_validate_source_evidence.py`: Python standard-library tests.
- `reports/`: local validation reports generated by the validator.

## Boundaries

This prototype must not:

- mutate source files,
- materialize source inventory artifacts outside reports,
- install packages,
- use non-stdlib dependencies,
- call GitHub/remotes,
- send externally,
- use local model/GPU processing,
- access credentials/account-security/customer/production systems,
- add runtime assistant behavior,
- package itself into governance core.

## Verification

```powershell
python -m unittest _bmad\memory\knx\runtime\optional-source-evidence-pack\tests\test_validate_source_evidence.py
python _bmad\memory\knx\runtime\optional-source-evidence-pack\scripts\validate_source_evidence.py
```

The validator reads `knx_storage_root` from `_bmad/config.user.yaml` by default. Use `--storage-root` only for explicit test or review scenarios.
