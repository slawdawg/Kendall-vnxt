# Story 9.3: Restore Provider Raw-Output UI Regression Coverage

Status: done

## Story

As Bob,
I want deterministic provider raw-output UI regression coverage restored,
so that dashboard and report surfaces keep proving they do not display or retain raw prompts, completions, provider payloads, secrets, or unnecessary source copies.

## Acceptance Criteria

1. Given dashboard or report surfaces render provider-adjacent evidence, when regression tests run, then they prove raw prompts, completions, provider payloads, secrets, and unnecessary source copies are absent from displayed evidence, and they verify bounded summaries, artifact references, or redacted metadata are used instead.
2. Given provider raw-output fixtures are needed for regression coverage, when tests are authored, then fixtures are synthetic, deterministic, local-only, committed or generated by an existing fake adapter, and do not require live provider calls, subscription credentials, external sessions, or network access, and fixture cases cover success, failure, empty, and oversized output states.
3. Given dashboard rendering fixtures are used, when 9.3 tests run, then fixtures are limited to dashboard, rendering, and raw-output UI/report cases, and they do not reuse 9.1 static drift fixtures or 9.2 subscription event-store fixtures as their source of truth.
4. Given a UI regression would expose raw provider output, when the regression suite runs, then the failing assertion identifies the affected UI/report surface and blocks the story from being considered done.
5. Given visible evidence is checked by regression tests, when the test runner validates the UI or report route, then assertions include DOM or text-state checks, not screenshots alone, and provider expansion, credential/session access, source mutation, launch retry automation, PR/merge/cleanup automation, failed-check bypass, and broad autonomy remain blocked.

### Architecture And Safety Boundaries

- Runtime evidence exports may include work item, event, attempt, route, workspace isolation, artifact reference, and summary metadata, but must not include broad filesystem snapshots or secret-bearing payloads. [Source: `docs/architecture/kendall-vnxt-worker-threat-boundary-2026-06-08.md#Artifact Boundary`]
- Output and artifact evidence must be bounded references and summaries, not raw stdout, stderr, generated patch content, raw prompts, raw completions, reasoning traces, provider payloads, secrets, or source snapshots. [Source: `docs/architecture/kendall-vnxt-process-lifecycle-design-2026-06-08.md#Output And Artifact Requirements`]
- Generic continuation language does not approve process launch, provider calls, command execution, source mutation by workers, network access, credential access, premium execution, external sends, or background runtime assistant behavior. [Source: `docs/architecture/kendall-vnxt-execution-authority-approval-checkpoints-2026-06-08.md#Language That Is Not Approval`]

- Do not reuse or mutate `scripts/fixtures/runtime-evidence-drift/known-good.json`, `scripts/fixtures/runtime-evidence-drift/known-bad.json`, or subscription launch event/idempotency fixtures as the source of truth.
